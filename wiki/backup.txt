You can help IndieWebCamp by downloading and keeping a backup of this wiki!

As of 2014-05-10 the wiki and images are about 70mb.

* Download [http://www.bittorrent.com/sync Bittorrent Sync]
* Create a folder on your computer or server to store the files
* Add a folder to btsync using this read-only secret: <code>BB3SRZEPU6B4EUMMD5XDWZBY2RDZSN33Q</code>

After adding the folder, your btsync client will begin downloading all the files!

[[User:Rascul.io|rascul]] has a mirror at [http://indiewiki.rascul.io http://indiewiki.rascul.io]. Easy to mirror it with wget or rsync:
: <code>wget -rm <nowiki>http://indiewiki.rascul.io</nowiki></code>
: <code>rsync -avz rsync://indiewiki.rascul.io:/indiewiki .</code>

[[User:kbsriram.com|KB]] has placed a [https://github.com/kbsriram/indiewebcamp-wiki-archive baseline archive at github] as of May11, 2014 and will update it ad-hoc. Current setup documented on Mac OS X below. (Why git? built-in consistency checking, poor-man's incremental updates and snapshots offers some protection against fat-fingering errors. File format is simple and likely to stick around for another decade (revisit in 2024? :) Simple distribution model, lots of clients, potentially easier to merge recoveries from multiple sources.)

# btsync client running in the background
$ crontab -l
45 10 * * 1,3,5 /Users/kbs/personal/indiewebcamp/backup-wiki.sh > /Users/kbs/personal/indiewebcamp/backup.log 2>&1
$ cat /Users/kbs/personal/indiewebcamp/backup-wiki.sh
#!/bin/bash

error_exit ()
{
    echo "$1"
    exit 1
}
date
cd /Users/kbs/personal/indiewebcamp/wiki-archive || error_exit "bad start"
changes=`git ls-files -m -o --exclude-standard`
if [ -z "${changes}" ]
then
    echo "no changes"
    exit 0
fi
/usr/local/bin/git add . || error_exit "unable to git-add"
/usr/local/bin/git commit -a -m "wiki-backup `date`" || error_exit "unable to git-commit"
/usr/local/bin/git push -u origin master || error_exit "unable to git push"
$ cat personal/indiewebcamp/wiki-archive/.git/config

[branch "master"]

== Archive Methods ==

=== rsync ===
<code>rsync -avz --delete --stats rsync://indieweb@indiewebcamp.com/indiewebcamp_wiki indiewebcamp</code>

<table><tr><td width="50%">
==== Pros ====
* Supports incremental changes
* Will properly delete local files when the remote files are deleted
</td><td>
==== Cons ====
* Requires running a separate publicly-accessible daemon on the server
</td></tr></table>

----
=== btsync ===
<table><tr><td width="50%">
==== Pros ====
* Keeps archive copies in sync nearly instantly
</td><td>
==== Cons ====
* Proprietary sync protocol
</td></tr></table>

----
=== wget ===
<code>wget --execute robots=off --no-parent --wait=1 --mirror https://indiewebcamp.com/wiki/backup/data</code>
<table><tr><td width="50%">
==== Pros ====
* Works over HTTP
* Supports incremental changes
</td><td>
==== Cons ====
* Local files won't be deleted when deleted on the server
</td></tr></table>

----
=== git ===
<table><tr><td width="50%">
==== Pros ====
* Works over HTTP
* Supports incremental additions and deletions
* Git repo stores full history
</td><td>
==== Cons ====
* Requires the server to take another step after exporting the wiki to text files, potentially more prone to errors
</td></tr></table>
